# PR Router

Сервис для управления Pull Request и автоматического назначения ревьюеров.

## Возможности

- создание команд и пользователей;
- создание Pull Request;
- автоматическое назначение до двух активных ревьюеров;
- получение PR пользователя;
- merge PR (идемпотентная операция);
- переназначение ревьюеров;
- массовая деактивация пользователей с безопасным переназначением ревьюеров;
- статистика по назначениям ревьюеров.

## Технологии

- FastAPI

## Запуск

1. Создайте файл `.env` на основе `.env.example`

2. Запустите проект:

```bash
docker compose up --build
```

После запуска:
- API доступно по адресу
```
http://localhost:8080
```

- Swagger
```
http://localhost:8080/docs
```

Во время запуска автоматически:
- запускается PostgreSQL
- применяются миграции Alembic
- запускается FastAPI

## Принятые решения

### Назначение ревьюеров

Ревьюеры назначаются случайным образом среди активных участников команды автора.

Если доступных пользователей меньше двух, назначается доступное количество.

## Merge

Операция merge является идемпотентной

Повторный вызов возвращает текущее состояние PR и не приводит к ошибке.

## Деактивация

При массовой деактивации выполняется попытка безопасного переназначения всех открытых PR.

Если подходящий кандидат отсутствует, текущий ревьюер остается назначенным, несмотря на статус `is_active = false`.
